# 包裹组件 contextenu

## 概述

这是一个用于,右键菜单的功能使用。该组件采用逻辑管理和渲染分开的方案。

## API 参考
| Prop        | 类型      | 必需 | 默认值     | 描述                                                |
| :---------- | :-------- | :--- | :--------- | :-------------------------------------------------- |
| `children`  | ReactNode | 是   | -          | 需要附加右键菜单的单个子组件。         |
| `menuItems` | Array     | 是   | `[]`       | 一个对象数组，用于定义菜单的结构和行为。|
| `onShow`    | Function  | 否   | `(metaInfo) => {}` | 菜单显示时触发的回调函数,参数是传输的metaInfo。             |
| `onHide`    | Function  | 否   | `(metaInfo) => {}` | 菜单隐藏时触发的回调函数,参数是传输的metaInfo。              |
| `className` | String    | 否   | `''`       | 应用于菜单根元素的额外 CSS 类名。        |
|`metaInfo`|    any | 否 | - | 如果需要使用渲染数据项的数据可传输|

## 实现原理
ContextMenu(触发器) → menuManager(管理器) → MenuHost(渲染宿主)
```javascript
  // 关键：确保全局只有一个 MenuHost 实例
  const manager = {
    host: null, // MenuHost 实例引用
    root: null, // React Root
    container: null, // DOM 容器

  ensureHostMounted() {
    if (this.root && document.body.contains(this.container)) {
    return; // 防止重复创建
  }
  // 创建 Portal 容器并挂载到 body
  this.container = document.createElement('div');
  document.body.appendChild(this.container);
  this.root = createRoot(this.container);
  this.root.render(<MenuHost onReady={...} />);
  }
  }
```
### 管理模式
```javascript
  // MenuHost 挂载后通过回调暴露接口
  useEffect(() => {
  onReady({ show, hide }); // 将控制权交给 manager
  }, [onReady, show, hide]);

  // manager 调用宿主方法
  showMenu(payload) {
  setTimeout(() => {
  this.host.show(payload); // 通过引用调用宿主方法
  }, 0);
  }
```
## 注意事项

- 默认使用iconfont 作为图标只需要传入名字即可,但是需要你导入对应的文件
- 多次渲染在一个组件内,同样需要符合key来进行处理
- 如果是嵌套的,你正常使用即可,因为他毕竟是单例的,会自动处理为传入的menuItem
- isNext事件:[需要确认环节] 只有一个参数参数是点击项数据[非DOM]
- 非isNext事件:[无需确认环节] 第一个参数是:事件对象e,第二个参数是代点击项数据[非DOM]
- 所有事件返回:返回当前传入的metaInfo项[不会传播event]
