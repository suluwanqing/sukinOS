# AlertConfirm 全局确认条组件文档

## 概述

AlertConfirm 是一个专为“确认操作”设计的全局通知条组件。它通常悬浮在页面顶部，用于提醒用户有未保存的更改、需要同步配置或执行关键性操作。与普通的 Alert 不同，它提供了双向交互按钮（确认/放弃）以及内置的加载状态逻辑，是复杂表单和设置页面的理想伴侣。

## 功能特性

- **非阻塞交互**：悬浮在页面顶部，不遮挡主体内容，允许用户在不关闭弹窗的情况下继续查看页面
- **内置加载状态**：确认按钮支持 loading 属性，点击后自动切换为旋转图标并禁用点击，防止表单重复提交
- **高度可配置**：支持自定义标题、按钮文字、图标以及确认按钮的主题类型
- **平滑动画**：采用贝塞尔曲线（Cubic-bezier）实现平滑的滑入滑出效果
- **Portal 渲染**：渲染至应用顶层，无视父组件的 `overflow: hidden` 或 `z-index` 限制

## 快速上手

### 使用

在需要监听状态变化的业务组件（如 Setting.jsx 或 LocalDev.jsx）中引入：
```javascript
import AlertConfirm from '@/component/alertConfirm/layout';
import React, { useState } from 'react';
import AlertConfirm from '@/component/alertConfirm/layout';
import SaveIcon from '@mui/icons-material/Save';
const MyComponent = () => {
    const [hasChanges, setHasChanges] = useState(false);
    const [isSaving, setIsSaving] = useState(false);
    const handleSave = async () => {
          setIsSaving(true);
          // 模拟异步保存请求
          await new Promise(resolve => setTimeout(resolve, 1500));
          setIsSaving(false);
          setHasChanges(false);
    };
    return (
        <div>
        {/* 业务内容 */}
            <AlertConfirm
            visible={hasChanges}
            loading={isSaving}
            title="检测到配置已修改，是否保存？"
            confirmText="保存并同步"
            icon={SaveIcon}
            onConfirm={handleSave}
            onCancel={() => setHasChanges(false)}
            />
        </div>
    );
};

```
## API 参考

### Props 属性

| 参数        | 类型                               | 默认值               | 说明                                                         |
| ----------- | ---------------------------------- | -------------------- | ------------------------------------------------------------ |
| visible     | boolean                            | false                | 是否显示组件                                                 |
| title       | string                             | '检测到未保存的更改' | 左侧显示的提示文本                                           |
| confirmText | string                             | '保存更改'           | 确认按钮的文字内容                                           |
| cancelText  | string                             | '放弃'               | 取消/放弃按钮的文字内容                                      |
| onConfirm   | function                           | undefined            | 点击确认按钮的回调函数                                       |
| onCancel    | function                           | undefined            | 点击放弃按钮的回调函数                                       |
| loading     | boolean                            | false                | 是否处于加载中状态。开启后确认按钮变为不可点击且显示加载动画 |
| icon        | ReactNode                          | null                 | 确认按钮左侧显示的图标（通常来自 Material Icons）            |
| confirmType | 'primary' \| 'danger' \| 'warning' | 'primary'            | 确认按钮的主题样式风格                                       |

## 实现原理

- **命名空间管理**：使用 `createNamespace('alert-confirm')` 生成 BEM 规范的类名（如 `su-alert-confirm__bar-content`），确保样式隔离
- **状态驱动动画**：通过监听 visible 属性切换 is-visible 类名，配合 CSS 中的 transform 和 opacity 实现从顶部滑入的效果
- **交互反馈**：按钮组件内置了针对 disabled 和 loading 状态的处理，通过 `backdrop-filter: blur` 增强在高背景复杂度下的易读性
- **性能优化**：使用 React.memo 包裹组件，只有在 Props 发生实质性变化时才会触发重渲染，避免在父组件输入时造成性能抖动

## 注意事项

- **容器定位**：默认采用 fixed 定位在页面顶部。如果应用内存在多个 fixed 元素（如导航栏），请确保该组件的 z-index 高于导航栏
- **Loading 策略**：建议在 onConfirm 触发异步操作前手动设置 loading 为 true，并在操作结束（无论成功或失败）后还原
- **单例与多例**：虽然该组件可以多次实例化，但由于其定位固定，建议在同一个视图内仅保留一个 AlertConfirm 实例，通过改变其 Props 来处理不同的确认逻辑
