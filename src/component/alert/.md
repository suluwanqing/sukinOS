# 全局 Alert 组件文档

## 概述

这是一个基于 React 的全局 Alert（警告提示）组件，采用命令式 API 设计。它允许在应用的任何地方（包括非组件代码）通过简单的函数调用（如 `alert.success()`）来显示提示信息，无需在组件树中手动管理状态。组件支持自动消失、多类型提示和自定义配置。

## 功能特性

- **全局调用**: 在任何地方均可触发 Alert。
- **多种类型**: 支持成功（success）、失败（failure）、警告（warning）等类型，每种类型有对应图标。
- **自动关闭**: 默认 1500ms 后自动消失，可配置持续时间。
- **多行支持**: 可配置多行显示以适应较长文本。
- **无障碍访问**: 通过 `role="alert"` 和 `aria-live` 属性提升可访问性。
- **Portal 渲染**: 使用 `createPortal` 渲染到 `body`，避免父组件样式干扰。


### 使用

在根组件（通常是 `App.jsx` 或 `main.jsx`）中引入 `AlertProvider`：

```jsx
import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App'
import AlertProvider from '@/component/alert/layout' // 根据你的路径调整

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <App />
    <AlertProvider /> {/* 确保 AlertProvider 在根层级渲染 */}
  </React.StrictMode>
)
```

### 在任何地方调用

现在，可以在任何 JavaScript 文件中导入并使用 `alert` 对象：

```jsx
import { alert } from '@/component/alert/layout' // 根据你的路径调整

// 在事件处理函数、API 回调、工具函数等中
const handleSubmit = async () => {
  try {
    // ... 一些操作
    alert.success('数据保存成功！', { duration: 2000 })
  } catch (error) {
    alert.failure('操作失败，请重试。', { duration: 2500, multiLine: true })
  }
}
```

## API 参考

### AlertProvider

无 props。只需在应用根组件中渲染一次即可。

### alert 对象

| 方法      | 参数                                                                   | 说明                                                        |
| :-------- | :--------------------------------------------------------------------- | :---------------------------------------------------------- |
| `success` | `(info: string, options?: { duration?: number; multiLine?: boolean })` | 显示成功提示。`info` 为提示信息，`options` 为可选的配置项。 |
| `failure` | `(info: string, options?: { duration?: number; multiLine?: boolean })` | 显示失败提示。`info` 为提示信息，`options` 为可选的配置项。 |
| `warning` | `(info: string, options?: { duration?: number; multiLine?: boolean })` | 显示警告提示。`info` 为提示信息，`options` 为可选的配置项。 |
| `dark` | `(info: string, options?: { duration?: number; multiLine?: boolean })` | 显示警告提示。`info` 为提示信息，`options` 为可选的配置项。 |


**options 参数:**

- `duration`: 提示显示的持续时间（毫秒），默认 1500ms。
- `multiLine`: 是否允许多行文本，默认 `false`。
- `allowMultiple`:'是否允许多个存在' ，默认  `false`开启后将会向下堆叠
## 实现原理

该组件基于 React 的 `createPortal`, `useImperativeHandle`, `forwardRef` 和 `useState`/`useEffect` Hooks 构建。

1.  **命令式调用**: 通过 `useImperativeHandle` 暴露方法给 ref，使得外部可以直接调用组件的显示逻辑。
2.  **状态管理**: 使用 `useState` 管理 Alert 的显示状态和配置。
3.  **自动隐藏**: 使用 `useEffect` 根据 `duration` 设置定时器，自动隐藏 Alert。
4.  **Portal 渲染**: 使用 `createPortal` 将 Alert 渲染到 `document.body`，避免受到父组件样式（如 `overflow: hidden`, `z-index`）的影响，确保其能显示在页面顶层。

## 注意事项

- 此组件为**单例**模式，同一时间只能显示一个 Alert。后调用的 Alert 会覆盖前一个。
- 确保 `AlertProvider` 已在应用根组件中渲染，否则 `alert` 方法调用将无效。
- 样式文件 (`style.module.css`) 需要自行实现以确保视觉效果。
- 图标的 SVG 路径可以根据你的设计需求进行替换。
