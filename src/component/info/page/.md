# GeneralPurposePage Component

一个通用的页面组件，用于快速创建如 404 Not Found、500 Server Error、空状态（No Data）等反馈页面。它高度可配置，允许通过 props 定制内容和行为。

## 使用方法

首先，导入组件：

```jsx
import GeneralPurposePage from '@/component/common/info/page/layout'; // 假设组件路径
```

然后，在你的应用中使用它：

```jsx
function My404Page() {
  return <GeneralPurposePage type="404" />;
}

navigate('/error-page?type=404&title=页面不存在&description=请检查 URL&actionText=返回首页&onActionClick=/');

//使用 state 传参
navigate('/error-page', {
    state: {
      type: 'no-data',
      title: '暂无内容',
      description: '当前没有找到相关数据',
      actionText: '刷新页面',
      onActionClick: '/home'
    }
});

// 使用 props
<GeneralPurposePage
  type="500"
  title="服务器错误"
  description="请稍后再试"
  actionText="重新加载"
  onActionClick={() => window.location.reload()}
  autoRedirect={{ enabled: true, path: '/', delay: 10 }}
/>

//混合使用
navigate('/error-page?type=404&title=自定义标题', {
  state: {
    description: '这是 state 传过来的描述',
    actionText: '去首页',
    onActionClick: '/dashboard'
  }
});

navigate('/error-page?type=404');
navigate('/error-page', {
    state: {
    type: '500',
      autoRedirect: {
        enabled: true,
        path: '/login',
        delay: 5
      }
    }
});

```

## Props

该组件的所有 props 均为可选。

| Prop | 类型 | 描述 | 默认值 |
| :--- | :--- | :--- | :--- |
| `type` | `string` | 预设的页面场景类型。可选值为：`'404'`, `'500'`, `'no-data'`。它会决定默认显示的图片、标题和描述。 | `'404'` |
| `imageUrl` | `string` | 自定义图片的 URL。如果提供，将覆盖 `type` 对应的默认图片。 | `undefined` |
| `title` | `string` | 自定义页面的主标题。如果提供，将覆盖 `type` 对应的默认标题。 | `undefined` |
| `description` | `string` | 自定义页面的描述文本。如果提供，将覆盖 `type` 对应的默认描述。 | `undefined` |
| `actionText`| `string` \| `null` | 操作按钮上显示的文本。如果设置为 `null` 或空字符串，按钮将不会被渲染。 | `'返回上一页'` |
| `onActionClick` | `function/string` | 操作按钮的点击事件处理函数。 | `() => window.history.back(),如果是string将会自动导航`  |
| `autoRedirect` | `object` | 自动重定向功能的配置对象。 | `undefined` |
| `autoRedirectEnabled` | `bolean` | 非对象传输下下自动重定向子配置。 | `false` |
| `autoRedirectPath` | `string` | 非对象传输下下自动重定向子配置。 | `/` |
| `autoRedirectDelay` | `number` | 非对象传输下下自动重定向子配置。 | `5` |

### `autoRedirect` 对象详解

当您需要启用自动跳转功能时，可以传入 `autoRedirect` 对象。该对象包含以下可选属性：

| 属性 | 类型 | 描述 | 默认值 |
| :--- | :--- | :--- | :--- |
| `enabled` | `boolean`| 是否启用自动重定向。设置为 `true` 时，倒计时将开始，并且操作按钮会被隐藏。 | `false` |
| `path` | `string` | 重定向的目标路径。 | `'/'` |
| `delay` | `number` | 从页面加载到执行重定向之间的延迟时间（单位：秒）。 | `5` |


## 示例

### 1. 默认的 404 页面

最简单的用法，所有配置均使用默认值。

```jsx
<GeneralPurposePage type="404" />
```

### 2. 启用自动重定向的 404 页面

5 秒后自动跳转到首页，并且不显示操作按钮。

```jsx
<GeneralPurposePage
  type="404"
  autoRedirect={{ enabled: true }}
/>
```

### 3. 自定义路径和延迟时间的重定向

10 秒后自动跳转到登录页面。

```jsx
<GeneralPurposePage
  type="404"
  autoRedirect={{
    enabled: true,
    path: '/login',
    delay: 10
  }}
/>
```

### 4. 500 服务器错误页面

使用 `500` 类型预设，并将按钮行为改为刷新页面。

```jsx
<GeneralPurposePage
  type="500"
  actionText="刷新页面"
  onActionClick={() => window.location.reload()}
/>
```

### 5. 完全自定义的 “暂无数据” 页面

覆盖所有内容，并且不显示任何操作按钮。

```jsx
import myImage from '@/assets/empty-box.png';

<GeneralPurposePage
  imageUrl={myImage}
  title="暂无订单记录"
  description="您还没有创建任何订单，快去下一单吧！"
  actionText={null}
/>
```
### 注意事项
- 只有props传输时onActionClick才被允许为函数传输,否则将跳转到/
