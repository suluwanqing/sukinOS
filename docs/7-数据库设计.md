# 数据库设计

## 一、数据库概览

| 数据库 | 类型 | 用途 |
|---|---|---|
| MySQL `react` | 主数据库 | 业务数据持久化存储 |
| Redis (DB 0) | 缓存 | Celery Broker |
| Redis (DB 1) | 缓存 | Celery Backend |
| Redis (DB 5) | 缓存 | 验证码、会话 |

## 二、MySQL 表结构

### 2.1 用户相关

#### `users` — 用户账户主表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | Integer | PK, AUTO_INCREMENT | 用户 ID |
| is_active | Boolean | default=True | 账号是否激活 |
| username | String(20) | | 用户名 |
| account | String(20) | UNIQUE | 登录账号 |
| password | String(20) | | 明文密码（开发/测试用） |
| hash_ps | String(80) | | bcrypt 哈希密码 |
| root | Boolean | default=False | 是否超级管理员 |
| email | String(25) | UNIQUE | 邮箱 |
| age | Integer | | 年龄 |
| name | String(8) | | 姓名 |
| sex | String(1) | | 性别 |
| phone | String(11) | | 电话 |
| address | String(25) | | 地址 |
| avatar | MEDIUMTEXT | | 头像 URL |
| permission | JSON | | 权限数据 |

**`permission` 字段结构**：
```json
{
  "role": "user",       // 角色标识 (root/user/developer)
  "first": 0,           // 一级权限位
  "second": 0,          // 二级权限位
  "keys": []            // 权限 Key 列表
}
```

**关联关系**：
```
users 1────1 user_expand          (扩展信息)
users 1────1 user_time_behavior   (时间行为)
users 1────N time_week            (周度时长汇总)
users 1────N sukinos_app          (上传的 APP)
```

#### `user_expand` — 用户扩展信息

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | Integer | PK, FK→users.id | 用户 ID |
| isonline | Boolean | default=False | 是否在线 |
| total_active_time | BigInteger | nullable | 累计活跃时长(秒) |

#### `user_time_behavior` — 用户时间行为

| 字段 | 类型 | 说明 |
|---|---|---|
| user_id | Integer | PK, FK→users.id |
| last_login | BigInteger | 最后登录时间戳 |
| d_mon ~ d_sun | JSON | 每日在线时长记录 |
| time_parts | JSON | 时段活跃度统计 |

**`d_xxx` 结构**：
```json
{
  "logout": 1700000000,
  "long": 3600
}
```

**`time_parts` 结构**：
```json
{
  "part_dawn": 5,
  "part_morning": 12,
  "part_afternoon": 8,
  "part_night": 3
}
```

#### `time_week` — 用户周度时长汇总

| 字段 | 类型 | 说明 |
|---|---|---|
| id | Integer | PK, AUTO_INCREMENT |
| user_id | Integer | FK→users.id, INDEX |
| d_mon ~ d_sun | Integer | 每日总时长(秒) |
| week_start_ts | BigInteger | 周一 00:00 时间戳, INDEX |

### 2.2 SukinOS APP 相关

#### `sukinos_app` — APP 文件表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| resource_id | String(120) | PK | APP 唯一标识 |
| user_id | Integer | FK→users.id NOT NULL | 上传者 |
| app_name | String(20) | | APP 名称 |
| created_at | DateTime | server_default=now() | 创建时间 |
| url | String(120) | NOT NULL | 下载地址 |
| is_private | Boolean | NOT NULL default=False | 是否私有 |
| size | Integer | | 文件大小(字节) |
| file_name | String(50) | NOT NULL | 存储文件名 |
| meta_info | JSON | | 元数据 |
| version | String(20) | NOT NULL default="0.1" | 版本号（乐观锁） |
| status | String(20) | NOT NULL default="under_review" | 状态 |
| audit_opinion | String(255) | nullable | 审核意见 |
| registry_enabled | Boolean | NOT NULL default=False | 是否注册权限池 |

**`status` 枚举**：
| 值 | 说明 |
|---|---|
| `under_review` | 审核中（初始状态） |
| `active` | 已上架 |
| `disabled` | 已下架/禁用 |

**`meta_info` 结构示例**：
```json
{
  "icon": "data:image/svg+xml;base64,...",
  "description": "一个记事本应用",
  "version": "1.0",
  "author": "开发者名"
}
```

**版本乐观锁**：`version` 字段用作 SQLAlchemy 的 `version_id_col`，每次更新自动调用 `increment_version()` 递增版本号（如 `0.9` → `0.10`）。

### 2.4 权限相关

#### `app_permission_registry` — APP 权限注册表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| resource_id | String(120) | PK | APP 资源标识 |
| permission_enabled | Boolean | NOT NULL default=False | 是否启用权限 |
| actor_rules | JSON | NOT NULL default={} | 权限规则 |
| created_at | DateTime | server_default=now() | 创建时间 |

**`actor_rules` 结构**：
```json
{
  "allowed_users": [1, 2, 3],
  "allowed_roles": ["developer"],
  "denied_users": [4],
  "denied_roles": ["user"]
}
```

#### `system_roles` — 角色定义表

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | Integer | PK, AUTO_INCREMENT | |
| name | String(50) | UNIQUE, NOT NULL | 角色标识 e.g. admin |
| label | String(100) | NOT NULL | 角色显示名 e.g. 管理员 |
| description | String(255) | default="" | 角色描述 |
| is_system | Boolean | default=False | 系统内置(不可删除) |
| created_at | DateTime | server_default=now() | |

**预置角色**：

| name | label | 说明 |
|---|---|---|
| `root` | 超级管理员 | 所有权限，不可删除 |
| `user` | 普通用户 | 默认注册用户，不可删除 |
| `developer` | 开发者 | 可管理自己的 APP |

## 四、Redis 数据结构

### 验证码缓存

```
Key: VerificationCode:{type}:{account}
Value: {"seed": "xxx", "code": "1234", "id": "uuid"}
TTL: 60-300s (依类型而定)

Key: VerifcationCodeLimit:{type}:{account}
Value: "1"
TTL: 60s (发送频率限制)
```

### Celery

```
Broker: redis://...:6379/0
Backend: redis://...:6379/1
```

## 五、索引策略

| request_logs | method | INDEX | 方法过滤 |
| request_logs | path | INDEX | 路径过滤 |
| request_logs | status_code | INDEX | 状态过滤 |
