# 系统架构

## 一、总体架构

SukinOS 采用**前后端分离**的单体 Web 应用架构，后端为 Python FastAPI 单体服务，前端为 React SPA，通过 RESTful API + WebSocket 通信。

## 二、请求处理流程

### 2.1 完整请求链路

```
用户请求
  │
  ▼
Nginx 反向代理 (端口 80/443)
  │ 静态文件 → 直接返回 dist/ 中的前端资源
  │ API 请求 → 转发到后端 127.0.0.1:8001
  ▼
CORSMiddleware (最外层)
  │ 检查 Origin，处理预检请求
  ▼
RoutePermissionMiddleware
  │ 1. 检查白名单（公开路径放行）
  │ 2. 解析 JWT Cookie 获取用户
  │ 3. root 用户无条件放行
  │ 4. 从 D_SystemConfig 读取路由权限配置
  │ 5. 检查 allowed_roles / allowed_users
  ▼
RequestLogMiddleware (纯 ASGI)
  │ 1. 根据配置决定是否记录日志
  │ 2. 收集请求体/响应体（脱敏 + 采样）
  │ 3. 解析操作用户信息
  │ 4. 将日志写入 request_logs 表
  ▼
StaticAuthMiddleware (最内层)
  │ 只有 /resource/sukinos/app/download 等静态路径才处理
  │ 1. 检查是否为受保护的静态资源
  │ 2. 验证 JWT / Refresh Token 静默刷新
  │ 3. 检查资源所有者或权限注册池
  │ 4. 放行后由 StaticFiles 返回文件
  ▼
路由处理器 (Route Handler)
  │ 可选: @RequireRoot / @RequirePermission 装饰器
  │ 可选: Depends(verify_auth()) 依赖注入
  │ 可选: Depends(require_root()) 依赖注入
  ▼
返回 Response
```

## 三、中间件详解

### 3.1 RoutePermissionMiddleware（路由权限中间件）
- **路径**: `Middleware/routePermission.py`
- **类型**: `BaseHTTPMiddleware`（Starlette）
- **功能**: 全局 API 路由权限检查
- **核心逻辑**:
  1. 白名单路径直接放行
  2. 已登录用户 → 检查 `check_route_permission()`（配置驱动）
  3. 未登录用户 → 检查路由是否需要认证
  4. root 用户全程放行

### 3.2 RequestLogMiddleware（请求日志中间件）
- **路径**: `Middleware/requestLog.py`
- **类型**: 纯 ASGI 中间件
- **功能**: 记录所有 API 请求的详细日志
- **特性**:
  - 敏感字段脱敏（password、token 等自动替换为 `***`）
  - JSON 响应体深度采样记录（12KB 截断 + 数组采样）
  - 二进制/流式响应只记录元数据
  - 支持动态开关（通过 log_config_cache 配置）
  - 自动解析操作人（Cookie JWT → 用户名/账号/ID）

### 3.3 StaticAuthMiddleware（静态资源鉴权中间件）
- **路径**: `Middleware/staticAuth.py`
- **功能**: 保护静态文件挂载点
- **处理的路径**:
  - `/resource/sukinos/app/download`（APP 文件，需鉴权）
  - `/media/user`（用户头像，完全公开）
- **APP 文件鉴权逻辑**:
  1. 查 `D_SukinosApp` 表判断文件是否私有
  2. 私有则需要身份验证（支持 Refresh Token 静默刷新）
  3. 验证通过：资源拥有者或权限注册池授权用户
  4. 静默刷新时自动注入新 Cookie

## 四、认证与鉴权体系

SukinOS 的权限认证分为 **5 个层级**，从全局到局部层层递进：

| 层级 | 实现方式 | 位置 |
|---|---|---|
| L1 - 全局中间件 | RoutePermissionMiddleware | 后端所有 API 路由 |
| L2 - 静态资源鉴权 | StaticAuthMiddleware | 静态文件挂载点 |
| L3 - FastAPI 依赖注入 | Depends(verify_auth()) | 路由处理器参数 |
| L4 - 装饰器 | @RequireRoot() / @RequirePermission() | 路由处理函数 |
| L5 - 前端中间件 | RoutePermission (axios 拦截器) | 前端 API 请求 |

详细说明见[权限体系详解](./权限体系详解.md)和[认证流程详解](./认证流程详解.md)。

## 五、后端技术架构

### 5.1 框架分层

```
Middleware          → 中间件（鉴权/日志/CORS）
    │
API Router Layer    → 路由定义（APIRouter）
    │                 验证参数（RequestModel Pydantic）
    │                 响应格式（api_response 统一包装）
    ▼
Service Layer       → Func/system/*（业务逻辑服务）
    │                 Func/judge/*（邮件/验证）
    ▼
Data Access Layer   → Model/DbModel/*（SQLAlchemy ORM）
                      Model/DbModel/default.py（引擎/会话管理）
```

### 5.2 响应格式统一

所有 API 响应通过 `api_response()` 函数统一包装（`Api/Re_Rs.py`）：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}
```

特性：
- 所有键名自动递归转为 camelCase
- 支持 bytes 类型直接返回二进制（如文件下载）
- HTTP 状态码智能映射（200/400/401/404/500）

## 六、前端技术架构

### 6.1 应用分层

```
main.jsx (入口)
  ├── Provider (Redux)
  │   └── PersistGate (状态持久化)
  │       └── AlertProvider / ConfirmProvider
  │           └── AuthorInfoPrinter
  │               └── RoutePermission (前端路由权限中间件)
  │                   └── RouterProvider
  │                       ├── / (首页/登录页)
  │                       ├── /sukinos (SukinOS 桌面)
  │                       │   ├── login (登录页面)
  │                       │   ├── deskbook (桌面环境)
  │                       │   └── resources/ (内置应用)
  │                       ├── /system (管理后台)
  │                       └── /about, /log 等
```

### 6.2 Redux Store 设计

```
store/
  ├── sukinos (持久化)        # SukinOS 桌面状态
  │   ├── userInfo            # 用户信息
  │   ├── theme               # 主题
  │   ├── assistant           # 辅助工具（验证码等）
  │   ├── setting             # 桌面设置
  │   ├── appStore            # 商店配置
  │   └── fileSystemConfig     # 文件系统配置
  ├── education (持久化)       # Education 应用状态
  └── 动态注入 (非持久化)      # 按需加载的 reducer
```

### 6.3 前端路由权限中间件

前端 `middleware/routePermission/` 在应用根级别加载路由权限配置，与 axios 请求拦截器联动，在发送 API 请求前本地拦截越权请求，避免无效的网络调用。

## 七、数据流

### 7.1 应用上传与发布流程

```
开发者上传 APP
  │ POST /sukinos/app/upload
  ▼
审核中 (status: under_review)
  │ 管理员审核
  ▼
已上架 (status: active) / 已下架 (status: disabled)
  │
  ├─ 公开 (is_private: false) → 商店可搜索、可安装
  └─ 私有 (is_private: true)  → 仅拥有者可见
       └─ 注册权限池 (registry_enabled: true) → 授权用户可见
```

### 7.2 Token 刷新流程

```
请求携带 Access Token
  │
  ├─ Access Token 有效 → 正常处理
  │
  └─ Access Token 过期
       │
       ├─ Refresh Token 有效 → 静默签发新 Token 对
       │   └─ 写入 Response Cookie → 前端透明
       │
       └─ Refresh Token 过期 → 返回 401 → 前端跳转登录页
```
